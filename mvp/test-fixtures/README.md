# Test Fixtures

Example projects for testing Kratai's code visualization features across different programming languages.

## Structure

```
test-fixtures/
├── typescript/    ✅ Supported - working examples
├── javascript/    🚧 Planned - coming soon
├── php/           🚧 Planned - coming soon
└── python/        🚧 Planned - coming soon
```

## Adding Test Fixtures

When creating new test projects:
1. Keep them minimal and focused
2. Include a README explaining what the example demonstrates
3. Commit the source code but not build artifacts or node_modules
4. Add `.gitignore` files as needed
